■ 프로필 페이지 html, css 
html, css파일을 따로 작성한 후 css 링크를 html 파일에 넣어주기로 했다. 
<link rel="stylesheet" href="../static/profile.css">

프로필 페이지에 들어가는 내용은 아래와 같다.

- 프로필 사진
(프로필 리스트)
- ID(닉네임)
- 반려동물 이름
- 반려동물 견종
- 반려동물 나이
(버튼)
- 프로필 수정
- 로그아웃

오늘 프로젝트를 진행하며 인상 깊었던 작업은 요소를 가운데로 정렬하는 코드였다. 
display : flex;
flex-direction : row;
justufy-content : center; 

위 작업을 진행할때 
-프로필 사진, (프로필 리스트) 내용을 container A로 묶어주고, (버튼) 내용들을 container B로 묶어줬다. 
처음에는 묶어줘야하는지 모르고 그냥 요소별로 코드를 작성하니 가운데 정렬과 양 옆으로 나뉘어지지 않았다. 

미니프로젝트를 함께하고있는 준호님이 6년차 개발자셨는데, 그림판을 활용해 왜 container로 묶어서 진행해야하는지 알려주셨다. 

이 경험을 통해 레이아웃을 구성할때 그림을 그려보며 진행하면 내가 묶어야할 div 코드를 확인하기 조금이나마 쉽다는것을 깨달았다. 


■ 프로필 수정 페이지 html, css 

프로필 수정 페이지는 프로필 페이지를 작성한 후 진행했기때문에 코드를 작성할때 아주 조금 시간을 단축할 수 있었다. 

프로필 수정 페이지에 들어가는 내용은 아래와 같다.

- 프로필 사진
(프로필 리스트)
- ID(닉네임)
- 반려동물 이름
- 반려동물 견종
- 반려동물 나이

프로필 사진은 부트스트랩을 활용했다. 
프로필 사진 크기 축소는 html img 코드에서 width, height를 활용했다. 

ul태그와 li태그는 다른 div에도 존재할 수 있기때문에 css를 작성할 경우 태그 종류에 따라 작성하고, ul태그와 li태그에 맞춰 작성해야한다. 

ex)
.profile_list ul{
    list-style:none;
    text-align: center;
    display: flex;
    flex-direction: column;
    align-items: center;
   }


.profile_list ul li{
    border: 2px solid;
    margin-top: 20px;
    padding: 5px;

    width: 10%;
}

만약 <div class="">가 아닌 <div id="">라면 .profile_list~~가 아닌 #profile_list~~로 작성해야한다.
